

# PRD_02 — 재료 기반 레시피 생성

## 1. 개요
PRD_01에서 추출된 재료 목록을 입력받아, `nvidia/nemotron-nano-12b-v2-vl:free` 모델(텍스트 모드)을 이용해 실행 가능한 레시피를 생성한다.

## 2. 목표
- 재료 목록을 받아 그 재료들로 만들 수 있는 레시피 여러 개를 추천한다.
- 각 레시피는 제목, 필요 재료, 조리 순서를 포함한 구조화된 형식으로 반환한다.

## 3. 범위

### In Scope
- 재료 배열을 입력으로 받는 레시피 생성 API
- 텍스트 모델 프롬프트 설계 및 구조화된(JSON) 응답 파싱
- 레시피 3~5개 추천, 각 레시피에 필요 재료/조리법 포함

### Out of Scope
- 사용자 선호도, 알레르기, 식단 제한 반영 (3단계 이후 확장 가능)
- 레시피 저장/북마크 (→ PRD_03)
- 영양 정보 계산

## 4. 사용자 플로우
1. PRD_01에서 인식된 재료 목록이 화면에 표시된다 (사용자가 수정 가능).
2. 사용자가 "레시피 추천" 버튼을 누른다.
3. 프론트엔드가 최종 재료 목록을 백엔드로 전송한다.
4. 백엔드가 재료 목록을 프롬프트에 포함해 모델에 요청한다.
5. 모델 응답을 파싱해 레시피 목록(JSON)으로 반환한다.
6. 프론트엔드가 레시피 카드 형태로 결과를 보여준다.

## 5. API 설계

### `POST /api/recipes/generate`
**Request**
```json
{ "ingredients": ["계란", "우유", "당근", "양파"] }
```

**Response** `200 OK`
```json
{
  "recipes": [
    {
      "title": "당근 오믈렛",
      "usedIngredients": ["계란", "당근", "양파"],
      "missingIngredients": ["소금", "식용유"],
      "steps": [
        "당근과 양파를 잘게 썬다.",
        "계란을 풀어 채소와 섞는다.",
        "팬에 식용유를 두르고 부친다."
      ],
      "estimatedCookTimeMinutes": 15
    }
  ]
}
```

## 6. 프롬프트 설계
```
아래 재료로 만들 수 있는 요리 레시피 3개를 추천해라.
각 레시피는 다음 JSON 형식의 배열로만 응답해라. 다른 설명은 포함하지 마라.

재료: 계란, 우유, 당근, 양파

형식:
[
  {
    "title": "요리명",
    "usedIngredients": ["사용된 재료"],
    "missingIngredients": ["추가로 필요한 재료"],
    "steps": ["조리 순서 1", "조리 순서 2"],
    "estimatedCookTimeMinutes": 숫자
  }
]
```

모델이 JSON이 아닌 텍스트로 응답할 경우를 대비해, 서버에서 JSON 파싱 실패 시 재시도(1회) 후 실패하면 에러를 반환한다.

## 7. 기술 의존성
- PRD_01의 `ingredients` 출력 형식
- `src/services/openrouterClient.js`의 `recognizeText(prompt)` 재사용 (또는 별도 `generateRecipes` 함수 추가)
- `config.js`의 모델/키 설정 재사용

## 8. 비기능 요구사항
- 레시피 생성 개수: 기본 3개
- 응답 시간: 15초 이내 목표
- JSON 파싱 실패율을 모니터링할 수 있도록 원본 응답 로깅(키 값은 로깅 금지)

## 9. 완료 기준 (Acceptance Criteria)
- [ ] 재료 목록 입력 시 실행 가능한 레시피 3개 이상 생성
- [ ] 각 레시피에 조리 순서가 최소 2단계 이상 포함
- [ ] 모델 응답이 JSON 형식이 아닐 때 서버가 크래시하지 않고 에러 처리
- [ ] 생성된 레시피가 PRD_03의 저장 데이터 구조와 호환됨

## 10. 오픈 이슈
- 재료가 너무 적을 때(예: 1개) 레시피 생성이 불가능한 경우 UX 처리
- 레시피 난이도/인분 정보 포함 여부
- 사용자가 특정 재료를 제외하고 재생성 요청할 수 있게 할지 여부
